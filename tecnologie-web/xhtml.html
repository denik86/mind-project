XHTML

XHTML è l'evoluzione del linguaggio HTML, rilasciato dal W3C il 26 Gennaio 2000. In particolare XHTML 1.0 è la versione successiva di HTML 4.01, quindi lo standard corrente.

La specifica di XHTML non introduce nuovi tag, ma viene definito una riformulazione di HTML come applicazione XML, quindi diventa uno standard più coerente e aiuta lo sviluppo di codice valido, facendo tornare a definire solo la struttura di una pagina web: un documento XHTML deve essere valido e ben formato (se inseriamo <font>, larghezza delle celle, ecc, il documento non è valido). Questo elimina parte dei problemi di presentazione di HTML (che si fa con i CSS) e, essendo un linguaggio XML, è interoperabile. Non avendo tag nuovi ed avendo le regole di XML è chiaro che XHTML risponda a due esigenze fondamentali:

  1. portare HTML nella famiglia XML con i benefici che ciò comporta in termini di estensibilità e rigore sintattico
  2. mantenere la compatibilità con i software che supportano HTML 4.0

XHTML elimina il problema del code forking perché supportato da diversi tipi di dispositivi (browser, browser per dispositivi mobili e screen reader) e un po anche dai vecchi browser.

L'uso di XHTML, e più in generale, degli standard web, porta quindi i seguenti vantaggi:

  • compatibilità con i browser
  • compatibilità con le future tecnologie
  • controllo centralizzato della presentazione
  • Indipendenza dal dispositivo
  • Migliore posizionamento nei motori di ricerca
  • Pagine leggere
  • Accessibilità
  • Migliore posizionamento sul mercato come sviluppatore

Versioni di XHTML

Esistono attualmente tre versioni disponibili e pubblicate di XHTML:

XHTML 1.0

Pubblicata il 26 Gennaio 2000. Consiste, come detto, nella riscrittura in XML di HTML 4.0 e si basa sulle tre DTD già definite per questo linguaggio:

  • XHTML Strict: è la forma più pura che aiuta a produrre codice in cui struttura e presentazione sono fortemente separati e rende più facile la modifica degli attributi di un elemento. Lo svantaggio è che non sempre è supportato bene dai vecchi browser. Non accetta i tag che sono stati definiti come deprecati in XHTML.
  • XHTML Transitional: è una forma transitoria creata per facilitare agli sviluppatori il passaggio ai nuovi standard. Accetta come validi anche i tag HTML che sono stati definiti come deprecati.
  • XHTML Frameset: nato per motivi di compatibilità per suddividere la finestra visualizzata dal browser in diversi frame, pratica ora deprecata dal W3C.

XHTML Basic

E' una versione ridotta del linguaggio, pensata specificatamente per dispositivi mobili: contiene solo gli elementi che si adattano a questi dispositivi (ad esempio esclude i tag frames).

XHTML 1.0

Basata sulla DTD Strict della versione 1.0, rappresenta la prima formulazione pratica del concetto di modularità. In questa versione, gli elementi fondamentali (i tag) sono raggruppati in una serie di moduli indipendenti. Secondo il W3C è la base della futura estensione di XHTML con altri set di linguaggi o moduli.

Le regole di sintassi XHTML

XHTML è un linguaggio XML quindi:

  • i tag e gli attributi sono case sensitive (tutto in minuscolo)
  • i tag devono sempre essere chiusi (anche se sono vuoti)
  • <br /> e non <br>
  • per compatibilità con i vecchi browser va usata la forma <p></p> per i tag non vuoti (anche se privi di contenuto) e <br /> per gli elementi vuoti
  • i tag devono essere aperti e chiusi nell’ordine corretto
  • l’ordine con cui si inseriscono gli attributi è irrilevante
  • i valori degli attributi vanno riportati tra "virgolette doppie"
  • tutti gli attributi devono avere un valore
  • un elemento in linea non può contenere un elemento di blocco

I browser cercano di visualizzare al meglio codice non valido, ma questo può portare ad interpretazioni arbitrarie.

Elementi che vengono ignorati

I browser ignorano completamente:

  • le interruzioni di linea non identificate con <br/> e non contenute in un tag <pre>
  • tabulazioni e spazi multipli
  • tag <p> nidificati ( <p><p> = <p> )
  • tag sconosciuti
  • commenti (ATTENZIONE: all'interno di un commento non è possibile inserire la stringa '--' (doppi trattini))

Struttura di base di un documento XHTML

Ecco come si presenta il codice di una semplice pagina XHTML basata sulla DTD Strict

Listato 1. Esempio di codice XHTML Strict 
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="it" lang="it">
  <head>
    <title>La mia prima pagina XHTML</title>
  </head>
  <body>
    <h1>Benvenuto!</h1>
    <p>Questo è il mondo di XHMTL!</p>
  </body>
</html>

Per poter verificare che questa pagina è valida e ben formata, è opportuno imparare a fare la convalida: Si va all'url http://validator.w3.org copiando il codice, caricando la pagina html oppure inserendo l'url della pagina e cliccare su "Validate this page". Se il documento è valido si riceverà una risposta di questo tipo: "Congratulations, this document validates as XHTML 1.0 Strict!".

Analizziamo ora, una per una, le quattro sezioni essenziali del documento:

Il prologo XML

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

La nostra pagina inizia con una rida di codice <?xml version="1.0"?> . La sua funzione è rendere esplicito il fatto che il documento è XML. Non è obbligatoria ma il suo uso è raccomandato dal W3C, per specificare:

  • la versione xml (version) obbligatoria
  • il tipo di codifica dei caratteri (encoding) opzionale
  • specificare che il documento non fa riferimento ad entità esterne (standalone="yes") opzionale

Purtroppo però molti browser non lo gestiscono correttamente, causando visualizzazioni scorrette o crash.

Dichiarazione di tipo documento

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

La dichiarazione del DOCTYPE (obbligatoria per il validatore), solitamente viene generata in modo automatico dall'editor HTML specifico. Ha il compito di informare il browser che si tratta di un documento html relativo alle specifiche e a quale delle tre DTD si conforma (in questo caso XHTML 1.0 Strict) e qual'è la lingua primaria.

Se non si specifica un doctype valido, i browser passano in modalità "quirks mode", ovvero una tecnica che consente ai browser di essere retrocompatibili con le pagine web che non sono conformi allo standard.
