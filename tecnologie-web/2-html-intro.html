HTML, XHTML e HTML5 - Introduzione

HTML e i browser

HTML (HyperText Markup Language) è un linguaggio per la costruzione di ipertesti. Viene definito come un linguaggio di markup (di marcatura), con cui si può indicare come i vari elementi vanno disposti in una pagina web e come questa pagina deve apparire nel browser web. Un documento html non è nient'altro infatti che un file di testo con delle indicazioni sul colore delle scritte, sulla posizione delle immagini all'interno della pagina, su come far scorrere il testo, e altre cose di questo genere.

Il Browser è il programma che si usa quando si naviga nel Web e svolge principalmente due compiti:

  1. scarica i vari files che si trovano su un computer remoto (il server) e che fanno riferimento a un certo indirizzo,
  2. legge i documenti scritti in html e, a seconda delle indicazioni ivi contenute, visualizza la pagina in un modo, piuttosto che in un altro; inoltre i vari files associati a quel documento (ad esempio le immagini, o i filmati in flash) vengono disposti secondo le indicazioni del codice html.
I files scaricati dal web vengono memorizzati in una particolare cartella del computer che prende il nome di chache. La visualizzazione di un file html da parte del browser prende il nome di rendering della pagina. Il motore di rendering è quella sezione del browser che si occupa di mostrare sul video la pagina.

L'evoluzione di HTML in XHTML

L'organizzazione che si occupa di standardizzare la sintassi del linguaggio HTML (il W3C: World Wide Web Consortium) ha rilasciato diverse versioni di questo linguaggio:

  1. Nel 1992 nasce la prima versione del linguaggio (non c'è ancora una descrizione del comportamento)
  2. Nel 1994 nasce HTML 3.0 che diventa uno standard
  3. Nel 1997 nasce il Recommendation HTML 4.01 del W3C
L'elasticità e la semplicità di HTML ha favorito lo svilupparsi di editor WYSIWYG. L'utilizzo di questi ultimi ha però portato a risultati graficamente molto accattivanti, ma di bassa qualità (tecnica del "costruisci adesso paga il prezzo più tardi"), ottenendo pagine non accessibili, spreco di banda e code forking (sviluppo di più versioni di pagie web per renderle interpretabili da browser diversi).

La guerra dei browser

HTML appartiene alla famiglia dei linguaggi SGML (un antenato di XML, ma non XML), ma è molto più semplice. Appena introdotto era troppo povero, non permetteva frame né immagini. Per accaparrarsi più utenti, i browser hanno cominciato ad inserire nel linguaggio nuovi tag proprietari, estendendone le funzionalità, ad esempio:

  • img (Netscape) vs object (MS Internet Explorer)
  • blink (Netscape): testo lampeggiante
  • marquee (Internet Explorer): testo scorrevole

La guerra dei browser ha portato un arricchimento delle possibilità offerte ma anche grossi problemi di incompatibilità. HTML 4.01 non risolve del tutto il problema, perchè utilizza tag che definiscono anche la presentazione e il layout.

XHTML 1.1 è essenzialmente una riformulazione della prima versione, in cui i tag e gli attributi vengono divisi in moduli per facilitarne l'uso.

Uno dei problemi principali di HTML è stata, quindi, la sua crescita disordinata e l'incompatibilità con vari browser. Inoltre il contenuto e aspetto non vengono considerati separatamente.

XHTML

XHTML è l'evoluzione del linguaggio HTML, rilasciato dal W3C il 26 Gennaio 2000. In particolare XHTML 1.0 è la versione successiva di HTML 4.01, quindi lo standard corrente.

La specifica di XHTML non introduce nuovi tag, ma viene definito una riformulazione di HTML come applicazione XML, quindi diventa uno standard più coerente e aiuta lo sviluppo di codice valido, facendo tornare a definire solo la struttura di una pagina web: un documento XHTML deve essere valido e ben formato (se inseriamo <font>, larghezza delle celle, ecc, il documento non è valido). Questo elimina parte dei problemi di presentazione di HTML (che si fa con i CSS) e, essendo un linguaggio XML, è interoperabile. Non avendo tag nuovi ed avendo le regole di XML è chiaro che XHTML risponda a due esigenze fondamentali:

  1. portare HTML nella famiglia XML con i benefici che ciò comporta in termini di estensibilità e rigore sintattico
  2. mantenere la compatibilità con i software che supportano HTML 4.0

XHTML elimina il problema del code forking perché supportato da diversi tipi di dispositivi (browser, browser per dispositivi mobili e screen reader) e un po anche dai vecchi browser.

L'uso di XHTML, e più in generale, degli standard web, porta quindi i seguenti vantaggi:

  • compatibilità con i browser
  • compatibilità con le future tecnologie
  • controllo centralizzato della presentazione
  • Indipendenza dal dispositivo
  • Migliore posizionamento nei motori di ricerca
  • Pagine leggere
  • Accessibilità
  • Migliore posizionamento sul mercato come sviluppatore

HTML 5

HTML 5 viene creato nel 2004, in risposta a XHTML 2, che sembrava fare le stesse cose della versione precedente del linguaggio in modo diverso, mentre si voleva qualcosa di più orientato alle applicazioni web. Il gruppo attivo che si occupa dell'HTML 5, oltre all'evoluzione generale di HTML, si chiama WHATWG (Web Hypertext Application Technology Working Group), coordinato da Ian Hickson e fondato nel 2004 da Apple, Mozilla e Opera, e ai quali si è aggiunto anche Google. Nel 2007 il gruppo ha chiesto al W3C che HTML 5 (che include al suo interno web apps 1.0 e web forms 2.0) diventi il punto di partenza dell'evoulzione dell'html e che la nuova generazione di specifiche html sia ufficialmente chiamata, appunto, HTML 5.

In risposta a tale richiesta il W3C crea il gruppo W3C HTML 5 Working Group che parte dalle specifiche prodotte dal WHATWG e abbandona l'XHTML 2.0.

Versioni di XHTML

Esistono attualmente tre versioni disponibili e pubblicate di XHTML:

XHTML 1.0

Pubblicata il 26 Gennaio 2000. Consiste, come detto, nella riscrittura in XML di HTML 4.0 e si basa sulle tre DTD già definite per questo linguaggio:

  • XHTML Strict: è la forma più pura che aiuta a produrre codice in cui struttura e presentazione sono fortemente separati e rende più facile la modifica degli attributi di un elemento. Lo svantaggio è che non sempre è supportato bene dai vecchi browser. Non accetta i tag che sono stati definiti come deprecati in XHTML.
  • XHTML Transitional: è una forma transitoria creata per facilitare agli sviluppatori il passaggio ai nuovi standard. Accetta come validi anche i tag HTML che sono stati definiti come deprecati.
  • XHTML Frameset: nato per motivi di compatibilità per suddividere la finestra visualizzata dal browser in diversi frame, pratica ora deprecata dal W3C.

XHTML Basic

E' una versione ridotta del linguaggio, pensata specificatamente per dispositivi mobili: contiene solo gli elementi che si adattano a questi dispositivi (ad esempio esclude i tag frames).

XHTML 1.1

Basata sulla DTD Strict della versione 1.0, rappresenta la prima formulazione pratica del concetto di modularità. In questa versione, gli elementi fondamentali (i tag) sono raggruppati in una serie di moduli indipendenti. Secondo il W3C è la base della futura estensione di XHTML con altri set di linguaggi o moduli.
