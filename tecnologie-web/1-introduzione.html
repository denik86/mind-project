La storia in breve

  • 1969: il Dipartimento della difesa Americana crea il primo nodo di ARPAnet all’UCLA. Lo scopo iniziale era creare una connessione tra vari sistemi. Contemporaneamente nasce anche l'ipertesto, ma inizialmente solo per scopi militari.
  • Anni ‘70-’80: nascita di numerose piccole reti per il trasferimento di file e posta elettronica. Sono per lo piu’ reti “private” (università e aziende).
  • 1986: viene creata NSFnet, una rete che collegava 5 università americane, che nel 1990 sostituisce ARPAnet per tutti gli usi non militari. Non c'è ancora il "web", ma semplici sistemi di scambio messaggi.
  • 1990: Tim Berners-Lee inventa il Web.
  • 1992: NSFnet collega circa un milione di computer in tutto il mondo e viene definita la prima versione dello standard HTML.
  • 1995: si inizia ad usare il nome Internet per la parte pubblica di NSFnet. Ci si rende conto che HTML contiene una capacità espressiva molto alta.

Internet

Internet (INTERconnected NETworks, ovvero reti (di reti) interconnesse) è un insieme di reti eterogenee e connesse in maniera lasca (cioè una rete che può "cadere"). Garantisce, attraverso il protocollo TCP/IP, una comunicazione affidabile ma non efficiente, cioè un servizio best effort: non è in grado di garantire l'inoltro di un messaggio ed un tempo massimo di arrivo (il tempo di caricamento di una pagina web non dovrebbe essere superiore ai 400 ms).

Un ISP (Internet Service Provider) è una struttura che fornisce il servizio di connessione agli utenti. I vari ISP sono a loro volta connessi ai fornitori di servizi di comunicazione di più alto livello.

Internet vs World Wide Web

E' importante ricordare che Internet e Web non sono la stessa cosa.

  • Internet è un'infrastruttura tecnologica che permette di far comunicare diversi computer (la rete fisica).
  • Il Web è l'insieme di software e protocolli installati sui diversi computer. In senso astratto è un insieme di documenti collegati tra loro.

HyperText Transfer Protocol

Il protocollo HTTP nasce al Cern, a Ginevra, da Tim Berners-Lee nel Giugno del 1999. Venne usato inizialmente per diffondere documentazione di tipo scientifico, in ambito bibliotecario.

L'HTTP stabilisce una connessione tra un client ed un server per lo scambio di documenti. Ogni documento è identificato univocamente da un URL (es. http://www.server.net/doc.html).

I collegamenti (link) possono essere:

  • interni ai documenti
  • fra documenti dello stesso server
  • fra documenti di server diversi

Curiosità

Prima dell'uscita di Windows 95, un documento HTML doveva avere l'estensione .htm, perchè i sistemi DOS e Windows 3.1 non erano in grado di gestire estensioni con più di 3 caratteri. Con i nuovi sistemi, le estensioni potevano avere una lunghezza superiore e succesivamente l'estensione di un file HTML è cambiata a .html (come jpg -> jpeg).

Comunicazione client / server

Richiesta dal cliente Web

Quando l'utente seleziona un indirizzo URL e lo invia al server, si genera un messaggio di questo tipo: 

GET //www.unipd.it/inf/index.html HTTP/1.1

Il metodo GET consente di ottenere il contenuto della risorsa indicata dall'URL. Quindi riceve un documento HTML e lo visualizza sullo schermo dell'utente intepretando i comandi di marcatura.

Altri metodi sono:

  • POST: esegue il documento specificato
  • HEAD: restituisce l'informazione relativa all'intestazione
  • PUT: sostituisce il documenti specificato con i dati allegati
  • DELETE: elimina il documento specificato

Risposta dal server Web

Quando il server riceve una richiesta da un browser (in ogni istante ne può ricevere molte contemporaneamente), si genera un messaggio di questo tipo:

http/1.1   200   OK

Dove:

  • http/1.1 è la versione HTTP
  • 200 è un codice di stato
  • OK è la spiegazione testuale

Dall'URL il server individua il documento da restituire, lo legge dal disco e lo ritrasmette al cliente.

La prima cifra del codice di stato può avere 5 valori:

Prima cifra Categoria
1 Informativa
2 Successo
3 Preindirizzo
4 Errore del client
5 Errore del server

La modalità appena descritta si applica in caso di pagine statiche, ma una pagina, o parte di essa, potrebbe essere dinamica: in questo caso il server crea "al volo" la pagina richiesta dall'utente sulla base dei dati forniti.

Il W3C

Il World Web Consortium (W3C) è un organismo indipendente che comprende le maggiori ditte produttrici di software per la rete, es. Google, Intel, AOL, Apple, Microsoft, Netscape, RealNetworks, molte università americane, ecc. Il sito web è: www.w3c.org.

Si occupa di proporre standard a largo spettro che comprendono un gran numero di tecnologie ed iniziative che riguardano il Web (HTML, XHTML, CSS, WAI, ...). Le proposte (Candidate Recommendation) vengono messe a disposizione su Web per raccogliere il numero più alto possibile di contributi. Successivamente diventano standard (Recommandation).

In genere il W3C mette a disposizione:

  • la definizione dello standard: recommandation
  • una suite di test per l'implementazione: test suite
  • un servizio di validazione

tutto offerto in modo gratuito proprio per promulgare l'utilizzo e la diffusione dello standard in oggetto.

L'alba del Web Design

All'inizio i designer provenivano dall'editoria cartacea, dove l'output cartaceo fisso (il giornale) può essere pensato come un file pdf (con qualche distinguo).

Nell'editoria su web entrano in gioco nuove variabili:

  • sistema operativo (tipi mime e font supportati, etc)
  • caratteristiche del dispositivo (schermo, connessione, etc)
  • browser (standard supportati, bug)
